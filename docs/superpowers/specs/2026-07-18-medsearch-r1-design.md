# MedSearch-R1 设计文档

## 1. 项目定位

项目名称：**MedSearch-R1：基于领域微调与成本感知强化学习的医学证据搜索 Agent**。

本项目面向医学选择题和可验证医学问答。模型先形成初始医学假设和置信度，再根据不确定性决定是否调用医学检索工具；获得证据后更新假设，最终提交答案及证据标识。项目不模拟真实医院检查，不声称提供临床诊疗能力。

项目主线复用现有学习阶段的成果：

- MedRAG 提供医学语料、BM25/Dense/Hybrid 检索器和检索评测；
- MedicalGPT 提供医学领域 LoRA/QLoRA SFT 能力；
- Search-R1 提供多轮工具交互、rollout、GRPO 和可验证奖励训练流程；
- LA-CDM 提供假设驱动、置信度校准和成本感知决策思想；
- R2RAG 的动态停止思想可作为后续对照或扩展，但不在第一版同时引入其全部模块。

## 2. 目标与成功标准

### 2.1 核心目标

验证领域 SFT 与成本感知搜索强化学习能否让 3B/4B 模型在医学问答中学会：

1. 对需要外部证据的问题主动检索；
2. 生成有效且不重复的医学搜索查询；
3. 根据检索证据更新答案和置信度；
4. 在证据充分时停止，避免无效多轮搜索；
5. 返回可核验的答案及证据文档标识。

### 2.2 完成标准

项目只有在以下条件同时满足时才算完成：

- Base、Medical SFT、固定单轮 RAG、ReAct Agent、Search-R1 风格 RL、完整 MedSearch-R1 均可由统一命令运行；
- 训练集、验证集和测试集严格隔离，并记录去重规则；
- 最终答案使用程序化指标评测，检索证据使用独立检索指标评测；
- 保存配置、随机种子、Git commit、样本量、显存峰值、训练时间和原始日志；
- 报告正确率、证据命中率、平均搜索轮数、重复查询率、非法调用率、延迟和 token 成本；
- 至少保留一组无效搜索、过早停止、过度搜索和高置信错误案例；
- 所有“提升”都明确相对基线、数据集和指标，不使用合成数据上的自评结果替代真实测试。

## 3. 范围与非目标

### 3.1 第一版范围

- 文本医学问题，以带标准答案的选择题为主；
- 本地医学语料检索，不依赖实时互联网；
- Qwen 3B/4B 级别模型；
- LoRA/QLoRA SFT；
- Search、Inspect、Submit 三种工具；
- 2～4 次最大工具轮数；
- 规则化、可验证的结果奖励；
- 单张 RTX 5090 32GB 上可运行的最小训练和评测流程。

### 3.2 第一版非目标

- 不模拟 CT、化验等患者级临床检查结果；
- 不使用 MIMIC-CDM，也不伪造患者轨迹；
- 不做多模态医学图像诊断；
- 不训练独立 LLM Judge 或大型过程奖励模型；
- 不同时加入 R2RAG、LinearRAG、Self-Play、Memory 等多个主要变量；
- 不对真实患者提供诊断建议，也不宣称达到临床部署标准。

## 4. 数据设计

项目需要四类数据资产，但不依赖 MIMIC-CDM。

### 4.1 医学 SFT 数据

用于 MedicalGPT 阶段，训练医学回答、格式遵循和基础领域能力。每条数据至少包含指令、输入、目标回答和数据来源。SFT 数据与最终测试问题必须进行题干和答案去重。

### 4.2 医学检索语料

优先复用 MedRAG 已验证的小型语料或官方语料子集。每个 chunk 保留稳定的 `document_id`、来源、标题和正文。必须记录语料规模、chunk 大小、overlap、索引类型和 embedding 模型。

### 4.3 RL 训练问题

第一版使用带标准答案的医学选择题，使最终答案能够程序化验证。训练样本包含问题、选项、标准选项和可选的证据标注。若没有证据标注，证据奖励只能使用检索命中或答案支持的保守代理指标，不得假装拥有金标准证据。

### 4.4 独立测试集

测试集不得用于 SFT、RL、提示调优或奖励系数选择。最终报告同时给出完整测试集结果，以及按“无需搜索、单跳搜索、多证据搜索”划分的子集结果。难度划分规则必须在查看最终结果前固定。

### 4.5 合成数据边界

合成轨迹可用于工具格式 SFT 和小规模冷启动；合成问题必须由标准答案或真实语料验证。最终核心结论必须来自独立的真实公开测试集，不允许由同一模型同时生成问题、答案和评判结果。

## 5. 系统架构

### 5.1 Agent 状态

每道题维护显式状态：

```text
question
options
current_hypothesis
confidence
search_history
evidence_documents
tool_round
terminated
```

状态必须可序列化，以便保存完整轨迹、重放失败案例和定位奖励错误。

### 5.2 工具接口

第一版只提供三个工具：

```python
search_medical_corpus(query: str, top_k: int) -> list[SearchResult]
inspect_document(document_id: str) -> Document
submit_answer(answer: str, evidence_ids: list[str], confidence: float) -> FinalAnswer
```

`search_medical_corpus` 通过统一 Retriever 接口调用 BM25、Dense 或 Hybrid 后端。Agent 不依赖检索器内部实现。

### 5.3 单题数据流

```text
医学问题
  → 生成初始假设与置信度
  → 选择 Search 或 Submit
  → Search 返回 Top-k 摘要和文档标识
  → 可选 Inspect 查看单篇证据
  → 更新假设、置信度和查询
  → 达到证据充分或轮数上限后 Submit
  → 记录答案、证据、成本和完整轨迹
```

轮数达到上限但没有合法 Submit 时，环境强制结束并标记 `forced_termination=true`，不得静默补写正确答案。

## 6. 训练方案

### 6.1 阶段 A：医学领域 SFT

使用 MedicalGPT 流程对 Qwen 3B/4B 做 LoRA/QLoRA。先验证格式遵循和医学 QA 行为，再将 adapter 合并或作为后续 RL 初始化。Base 与 SFT 必须在同一测试子集上对比。

### 6.2 阶段 B：工具格式冷启动

使用少量经过验证的轨迹教模型输出合法工具调用。该阶段只解决 JSON/标签格式和基本调用顺序，不将合成轨迹的质量当成最终能力提升。

### 6.3 阶段 C：Search-R1 风格 GRPO

在医学问题与本地检索环境上采样多轮轨迹，以最终答案正确性为主奖励。第一轮只使用答案、格式、非法调用和重复查询奖励，跑通后再单独加入证据与搜索成本奖励。

### 6.4 阶段 D：假设、置信度与成本控制

引入 LA-CDM 思想，要求 Agent 在每轮维护假设和置信度，并加入搜索成本。置信度校准作为独立实验，不与第一轮 GRPO 同时调参。

奖励结构固定为以下形式；各系数不是预设实验结论，必须在验证集上逐项确定：

```text
R = R_answer
  + λ_format R_format
  + λ_evidence R_evidence
  - λ_search C_search
  - λ_repeat C_repeat
  - λ_invalid C_invalid
```

奖励系数必须通过验证集和奖励分量日志确定。每次实验只改变一个主要奖励分量，并检查模型是否出现“不搜索直接猜”“反复改写同义查询”或“为了证据奖励复制文档标识”等 reward hacking。

## 7. 单卡 32GB 预算

第一版硬件配置目标：

| 配置项 | 起步设置 |
|---|---|
| Backbone | Qwen2.5-3B 或同级 3B；稳定后再尝试 Qwen3-4B |
| 微调 | QLoRA 4-bit 或经实测可行的 BF16 LoRA |
| 最大上下文 | 2048～4096 tokens |
| 最大工具轮数 | 2，稳定后增加到 4 |
| 每题 rollout | 2，显存允许后尝试 4 |
| micro batch | 1 |
| 梯度累积 | 根据有效 batch 调整并记录 |
| Gradient Checkpointing | 开启 |
| 检索服务 | 优先 CPU/独立进程，避免与训练争抢显存 |

任何显存和训练时间数字都必须由实际运行记录。遇到 OOM 时按既定顺序缩短序列、减少 rollout、减小 batch、确认梯度检查点和量化，再考虑更换模型。

## 8. 基线与实验矩阵

| 系统 | 医学 SFT | 检索工具 | RL | 假设/置信度 | 成本奖励 |
|---|---|---|---|---|---|
| Base | 否 | 否 | 否 | 否 | 否 |
| Medical SFT | 是 | 否 | 否 | 否 | 否 |
| Fixed RAG | 是 | 固定单轮 | 否 | 否 | 否 |
| ReAct Agent | 是 | 动态多轮 | 否 | 可选提示 | 否 |
| Medical Search-R1 | 是 | 动态多轮 | GRPO | 否 | 否 |
| MedSearch-R1 | 是 | 动态多轮 | GRPO | 是 | 是 |

主要指标：

- QA Accuracy / Exact Match；
- Recall@k、MRR 或证据命中率；
- 平均搜索轮数和平均召回文档数；
- 重复查询率、非法调用率、强制停止率；
- 平均延迟、token 消耗和显存峰值；
- 有标注条件下的 ECE 或 Brier Score；
- 简单题与复杂题的分组表现。

最终分析必须回答：提升来自医学 SFT、检索、RL，还是更高的搜索成本；如果没有提升，应定位到领域能力、查询生成、召回、证据利用、停止策略或奖励设计中的具体瓶颈。

## 9. 错误处理与轨迹审计

- 非法 JSON、未知工具和缺失参数返回结构化错误，并计入非法调用率；
- 空检索结果返回合法空列表，不伪造证据；
- 重复查询以规范化后的 query 哈希检测，同时保存原始 query；
- 无效 `document_id` 返回明确错误，不自动替换成其他文档；
- 超过最大轮数强制结束并记录原因；
- 每条轨迹保存问题 ID、模型版本、检索器配置、动作、观察、奖励分量和最终输出；
- 日志不得包含受限制或不应公开的原始敏感医疗数据。

## 10. 验证策略

### 10.1 单元验证

- 工具调用解析与 schema 校验；
- 答案标准化和选择题判分；
- 重复查询检测；
- 搜索成本累计；
- 最大轮数与强制停止；
- 奖励各分量的独立计算。

### 10.2 集成验证

- 使用 5～10 道固定题完成 Search → Observe → Submit 闭环；
- 检查 Retriever 切换不改变 Agent 工具协议；
- 检查完整轨迹可保存和重放；
- 在启动训练前先运行零样本和 SFT 推理基线；
- 用极小训练集验证 loss、reward、显存和 checkpoint，再扩大数据。

## 11. 主要风险与缓解

| 风险 | 缓解措施 |
|---|---|
| 医学 SFT 数据泄漏到测试集 | 题干、答案和语义近重复检查；固定独立测试集 |
| 检索语料直接包含测试答案模板 | 报告语料来源并做文档级泄漏审计 |
| 奖励鼓励不搜索直接猜 | 分题型统计搜索行为；单独调节成本奖励 |
| Agent 引用无关证据 | 独立评测证据命中和答案正确率 |
| 合成轨迹形成闭环自证 | 合成数据仅作冷启动；真实测试集最终评估 |
| 单卡 rollout OOM | 3B、QLoRA、rollout=2、短上下文逐步扩展 |
| 医疗措辞被误解为临床产品 | README 明确研究用途和非临床声明 |

## 12. 与学习路线的关系

当前仍从阶段 1 MedRAG 开始，不因最终项目设计而跳过基础阶段。建议后续路线为：

```text
阶段 1～3：MedRAG / R2RAG / LinearRAG
  → 形成医学检索工具和评测基础
阶段 4：MedicalGPT LoRA/QLoRA SFT
  → 形成医学领域初始化模型
阶段 5：Search-R1 3B
  → 跑通搜索环境、rollout、reward 和 GRPO
阶段 6：MedSearch-R1
  → 完成领域 SFT、医学检索与成本感知 Agentic RL 的组合实验
```

只有在前三个来源模块分别跑通并有真实基线后，才进入组合阶段，避免无法判断最终结果来自哪个模块。
